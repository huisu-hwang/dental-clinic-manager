# 네이버 블로그 자동 작성 가이드라인

> 출처: "대표님을 위한 상업블로그 운영법" (이타인/William Kyeung) 기반
> 하얀치과 블로그 마케팅 자동화 시스템에 적용

---

## 1. 콘텐츠 전략

### 글 유형 비율
- **정보성 글 : 홍보성 글 = 2 : 1** (필수)
- 홍보글만 올리면 방문자 감소 + 노출 패널티 발생
- 정보성 글이 체류시간이 길고 (평균 1분 30초+), 꾸준한 유입 보장

### 글 유형 정의
- **정보성 글**: 사람들이 검색해서 찾아보는 HOW-TO 콘텐츠 (치아 관리법, 스케일링 주기 등)
- **홍보성 글**: 치과 소개, 시술 안내, 이벤트 등 직접적인 병원 홍보
- **일상 글**: 원장/스태프 일상, 병원 에피소드 (블로그 신뢰도 향상용)

### 치과 특수 주의사항 (의료법)
- **의료인이 아닌 사람이 병원 홍보 글을 쓰는 것 자체가 불법** (의료법 위반)
- 치과 블로그는 **치과의사 본인 명의** 또는 **일반 정보성 콘텐츠** 위주로 운영
- 시술 후기, 효과 등 직접적 광고 표현 주의

## 2. 키워드 전략

### 키워드 선정 원칙
1. **검색도구 사이트에서 반드시 키워드 조사 후 글 작성**
   - 키워드마스터 (whereispost.com/keyword)
   - 블랙키위 (blackkiwi.net)
   - 리얼키워드 (realkeyword.co.kr)
   - 네이버 데이터랩 (datalab.naver.com)

2. **로얄 키워드 공략** (황금키워드보다 중요)
   - 꾸준한 검색량 있을 것
   - 월간 발행량이 적을 것
   - 정보성 콘텐츠일 것
   - **시의성(최신성)이 가장 중요** - 네이버는 최근 발행 글 우대

3. **블로그 지수에 맞는 키워드 선택**
   - 초반: 월 검색량 1,000~3,000 정보성 키워드 공략
   - 성장기: 점차 경쟁 키워드로 확장
   - 검색량 높은 키워드는 최적화 블로그들이 이미 차지

4. **연관검색어 활용**
   - 네이버 자동완성/연관검색어가 검색량이 높은 이유: 사람들이 클릭으로 검색
   - 연관검색어를 제목에 활용하면 유리

### 키워드 운영 규칙
- 여러 키워드를 돌아가며 사용
- **하나의 키워드가 상위노출 성공 시 → 같은 키워드로 글쓰지 않기** (어뷰징 판정 위험)
- 지역+업종 키워드는 경쟁률 높지만 실제 고객 전환률 높음
- 다양한 키워드로 확장성 넓히기

### 치과 키워드 예시 (정보성)
- 치아 관리, 스케일링 주기, 잇몸 건강, 치실 사용법
- 임플란트 종류, 교정 기간, 미백 방법
- 치아 통증 원인, 사랑니 발치 후 관리
- (주의: "최저가", "최고", "추천" 등 상업 키워드 사용 금지)

## 3. 제목 작성 규칙

### 필수 원칙
1. **제목에 검색 키워드 반드시 포함**
2. **키워드는 제목 앞쪽에 배치**
3. **단어들이 떨어지지 않게 연결**
4. **서브키워드 최소화** (키워드 많으면 노출 분산)
5. **제목은 간결하게** (길고 서브키워드 많으면 상위노출에 불리)

### 좋은 제목 vs 나쁜 제목
```
나쁨: 스케일링잘하는곳 - 고급스런 강남치과에서 전문의가 직접 시술하는 프리미엄 스케일링
좋음: 스케일링 주기 적정 횟수는? 치석 제거 후 주의사항~
```
- 나쁜 예: 키워드 여러개 나열, 홍보성, 길고 부자연스러움
- 좋은 예: 핵심 키워드 1개, 자연스러운 문장, 정보성

### 제목 수정 금지
- 발행 후 제목과 키워드 수정은 검색 순위에 악영향

## 4. 본문 작성 규칙

### 네이버 SEO 최적화 글쓰기
- **글쓰기 전 검색도구에서 키워드 확정**
- **키워드는 본문 내 자연스럽게 적절히 배치** (남발 시 순위 하락)
- **글자수 최소 1,000자 이상** (띄어쓰기 제외, 충분한 정보량)
- **키워드 검색 후 상위 노출 글 3개 분석 후 작성** (벤치마킹)
- 문단별 이미지 배치 (텍스트만 나열하지 않기)

### 글쓰기 톤 & 스타일
- **쉬운 언어 사용**: 전문 용어 사용 시 반드시 괄호로 풀이 (예: "치주낭(잇몸과 치아 사이의 틈)")
- **친절한 설명체** (하우투 글은 사람들이 잘 안 쓰기 때문에 경쟁 적음)
- **실제적인 도움이 되는 내용**: 읽고 나서 바로 실천할 수 있는 구체적 정보 제공
- **개인적이고 독창적인 글** 선호 (네이버가 좋아하는 글)
- **전문적이면서 독창적인 정보성 글** 선호
- **일방적인 홍보글, 상업적인 글** 비선호

### 논문 인용을 통한 신뢰성 확보 (옵션 기능)
글 생성 시 `useResearch: true` 옵션으로 활성화.

**작동 흐름:**
```
키워드/주제 확정
→ 학술 논문 검색 (Google Scholar, PubMed, KCI 등)
→ 관련 논문 2~3편 선별
→ 핵심 연구 결과를 본문에 자연스럽게 인용
→ 글 하단에 출처 표기
```

**논문 검색 소스:**
| 소스 | 용도 | API |
|------|------|-----|
| Google Scholar | 국내외 논문 통합 검색 | SerpAPI / Scholarly |
| PubMed | 의료/치과 논문 | NCBI E-utilities |
| KCI (한국학술지인용색인) | 국내 논문 | KCI Open API |
| DBpia / RISS | 국내 학술 DB | 웹 스크래핑 |

**인용 규칙:**
- 논문 내용을 그대로 복붙하지 않고, **쉬운 말로 풀어서 설명**
- 인용 시 자연스럽게 녹이기: "○○대학 연구팀의 연구(2024)에 따르면..."
- 글 하단에 출처 목록 표기: `[참고 논문] 저자명, "논문 제목", 학술지명, 연도`
- 치과 관련 글에 논문 인용 시 C-Rank 전문성 점수에 긍정적 영향

**인용 예시 (치과 글):**
```
잇몸 건강을 위해서는 하루 2번 이상 칫솔질이 권장됩니다.
서울대학교 치의학대학원 연구팀(2023)에 따르면, 하루 3회 칫솔질을
하는 그룹이 1회 그룹에 비해 치주질환 발생률이 47% 낮았습니다.

[참고 논문]
- 김○○ 외, "칫솔질 빈도와 치주질환의 상관관계 분석", 대한치과학회지, 2023
```

### 팩트체크 기능 (옵션 기능)
글 생성 후 `factCheck: true` 옵션으로 활성화.

**작동 흐름:**
```
AI가 글 생성 완료
→ 본문에서 사실 주장(수치, 통계, 의학적 주장) 추출
→ 각 주장을 신뢰할 수 있는 소스와 대조
→ 검증 결과 리포트 생성
→ 부정확한 내용 자동 수정 또는 플래그 표시
```

**검증 소스:**
| 소스 | 검증 대상 |
|------|----------|
| 대한치과의사협회 | 치과 시술/치료 정보 |
| 국민건강보험공단 | 보험 적용, 비용 정보 |
| 학술 논문 DB | 의학적 수치/통계 |
| 식약처 | 의료기기/약품 정보 |
| 건강보험심사평가원 | 진료비/시술 통계 |

**검증 등급:**
```typescript
interface FactCheckResult {
  claim: string;           // 검증 대상 문장
  verdict: 'verified' | 'unverified' | 'incorrect' | 'outdated';
  source?: string;         // 검증 출처
  suggestion?: string;     // 수정 제안 (incorrect일 때)
  confidence: number;      // 검증 신뢰도 (0~1)
}
```

- `verified`: 신뢰할 수 있는 출처로 확인됨
- `unverified`: 확인 가능한 출처를 찾지 못함 → 해당 문장에 "~로 알려져 있습니다" 등 완화 표현 적용
- `incorrect`: 부정확한 정보 → 자동 수정 또는 삭제
- `outdated`: 오래된 정보 → 최신 데이터로 업데이트

**치과 글 특별 검증 항목:**
- [ ] 시술 기간/비용이 현실적인 범위인가
- [ ] 의학적 효과가 과장되지 않았는가 (의료법 위반 주의)
- [ ] 약품/재료명이 정확한가
- [ ] 보험 적용 여부가 최신 기준인가

### 글 어투 & 분위기 선택 (사용자 설정)
글 생성 시 `tone` 옵션으로 선택 가능:

| 옵션값 | 어투 | 특징 | 적합한 글 유형 |
|--------|------|------|------------|
| `friendly` | 친근한 반말체 | "~해요", "~거든요", 이모티콘 가능 | 일상글, 가벼운 정보글 |
| `polite` | 정중한 존댓말 | "~합니다", "~드립니다" | 전문 정보글, 치과 소개 |
| `casual` | 구어체 | "~인데요", "~더라고요", 대화하듯 | 후기형 정보글 |
| `expert` | 전문가 톤 | "~입니다", 논문 인용, 데이터 기반 | 의료 정보, 연구 기반 글 |
| `warm` | 따뜻한 공감체 | "~셨죠?", "걱정되시죠?", 위로/공감 | 환자 불안 해소 글 |

**사용 예시:**
```typescript
interface BlogGenerateOptions {
  topic: string;
  keyword: string;
  tone: 'friendly' | 'polite' | 'casual' | 'expert' | 'warm';
  useResearch: boolean;    // 논문 인용 여부
  factCheck: boolean;      // 팩트체크 여부
}

// 예: 임플란트 정보글을 따뜻한 공감체로, 논문 인용 포함
const options: BlogGenerateOptions = {
  topic: '임플란트 시술 후 관리법',
  keyword: '임플란트 관리',
  tone: 'warm',
  useResearch: true,
  factCheck: true,
};
```

**어투별 같은 내용 비교:**
```
[friendly] 스케일링 아프지 않아요! 생각보다 금방 끝나거든요~

[polite]   스케일링은 통증이 거의 없으며, 약 30분 내에 완료됩니다.

[casual]   스케일링 해봤는데요, 솔직히 별로 안 아프더라고요.

[expert]   스케일링(치석 제거술)은 국소마취 없이 시행 가능하며,
           평균 시술 시간은 20~30분입니다.

[warm]     스케일링이 아플까 봐 걱정되시죠? 괜찮아요.
           대부분의 분들이 "생각보다 안 아프다"고 말씀하세요.
```

### 금지 표현/키워드 (상업 키워드)
네이버가 콘텐츠 점수에서 불이익을 주는 키워드:
> 후기, 효과, 효능, 카드, 추천, 최초, 최저, 최대, 최고, 체험, 제일, 저렴한곳, 완전, 엄청, 신용, 무료, 공짜, 가장, 가입, 1위

### 금지 행위
- 뉴스/외부 글 복사 붙여넣기
- 같은 키워드 어색하게 반복
- 글마다 같은 외부링크 삽입
- 본문에 전화번호/주소 반복 기입
- 해시태그 10개 이상 과다 사용
- 특정 단어 10번 이상 과도하게 사용

## 5. 이미지 규칙

### 필수 원칙
- **직접 찍은 사진일수록 좋음** (AI 생성 이미지는 "직접 찍은" 것은 아님 → 독창성 중요)
- **글 주제와 연관된 이미지** 사용
- **이미지마다 다른 고유한 이미지** 사용 (재사용/비슷한 사진 금지)
- 이미지에 홍보문구 넣지 않기 (OCR로 감지됨)

### AI 생성 이미지 사용 시 주의
- 매번 다른 고유한 이미지 생성 (동일 프롬프트 반복 금지)
- 이미지에 텍스트(글씨) 과다 삽입 금지
- 홍보성 문구가 포함된 이미지 금지
- 무료 이미지 사이트의 뻔한 스톡 사진 지양

### 무효 이미지 (네이버가 필터링하는 이미지)
- 글씨가 많은 이미지
- 홍보 문구가 있는 이미지
- 무료 스톡 이미지 (이미 많이 사용된 것)
- 개인정보 포함 이미지
- 성적/폭력적 이미지

### 이미지 배치
- 문단마다 1~2장 적절히 배치
- 글 전체에 3~7장 정도 사용
- 이미지만 있는 짧은 글 대량 발행 금지 (지수 악영향)

## 6. 발행 규칙

### 발행 빈도
- **하루 최대 3개 포스팅 권장** (기계적 대량 생산 = 패널티)
- 1일 1포스팅이 이상적
- 꾸준히 매일 발행하는 것이 중요 (C-Rank 상승)

### 발행 후 검색 반영 과정
1. **1차 반영 (1~2시간)**: 영상/이미지 먼저 노출, 블로그 최신순 노출
2. **2차 반영 (3~4시간)**: 순위 거의 결정, 100위 내 진입 시 상위노출 가능성 높음
3. **3차 반영 (일정기간)**: 방문자 체류시간/반응 기반 순위 조정

### 카테고리 관리
- 업체 소개 카테고리 필수 생성
- 대표글 지정 (인기글 대신)
- 전문 주제에 50% 이상 게시물 집중 (C-Rank / 인플루언서 선정)

## 7. 블로그 건강 관리

### 블로그 지수 체계
```
저품질(스팸) → 일반 → 준최적화(1~6단계) → 최적화(1~3단계)
```

### C-Rank (씨랭크)
- **맥락(Context)** + **내용(Content)** + **연결된 소비/생산(Chain)** + **출처의 신뢰도(Creator)**
- 전문 주제를 지속적으로 쓰면 해당 키워드에서 상위권 가능
- 치과 관련 글만 꾸준히 쓰면 치과 키워드 C-Rank 상승

### 저품질(스팸) 블로그 방지
- 홍보성 글 비율 과다 금지
- 같은 키워드 반복 사용 금지
- 외부 이미지/글 복사 금지
- 외부링크 과다 삽입 금지
- 기계적 대량 생산 금지

### 누락 방지
- 위 규칙을 어기면 검색 노출 자체가 안 됨 (검색 누락)
- 네이버 OCR(이미지 문자 인식) 필터링 강화 → 이미지 내 텍스트 주의
- 정보성 키워드에 홍보글 쓰면 **뷰누락** 발생

---

## 8. 홍보 포스팅 작성법 (정보성 글과 별도)

### 올바른 홍보 방법
- 매번 같은 홍보 이미지/링크 삽입 금지
- **내부링크** 활용 (소개글로 연결) → 외부링크보다 블로그 점수에 도움
- 홍보 포스팅에만 전화번호/홈페이지 1번 삽입 가능
- 대표 소개 포스팅 1개 만들고, 다른 글에서 내부링크로 연결

### 이미지 누락 방지 (라이브러리 가리기)
- 홍보 이미지를 한 게시물에 올리고
- 다른 게시물에서 복사/붙여넣기로 삽입 (라이브러리에 미포함)
- 이미지 검색에서 누락 여부 확인: 제목에 쌍따옴표 붙여 이미지 검색

## 9. 자동화 시스템 적용 규칙 요약

### AI 글 생성 시 체크리스트

## 10. 병원 공지글 & 임상글 생성

### 10-1. 글 유형 확장

기존 정보성/홍보성 외에 2가지 유형 추가:

```typescript
type BlogPostType =
  | 'informational'   // 정보성 글 (AI 생성 + AI 이미지)
  | 'promotional'     // 홍보성 글 (AI 생성 + AI 이미지)
  | 'notice'          // 병원 공지글 (템플릿 기반 + 병원 사진)
  | 'clinical';       // 임상글 (임상 사진 활용 + AI 본문 보조)
```

### 10-2. 병원 공지글 (Notice)

진료 일정 변경, 이벤트, 신규 장비 도입 등 병원 운영 관련 공지.

**템플릿 종류:**

| 템플릿 | 용도 | 예시 |
|--------|------|------|
| `holiday` | 휴진/연휴 안내 | "설 연휴 진료 안내" |
| `schedule` | 진료시간 변경 | "7월 진료시간 변경 안내" |
| `event` | 이벤트/프로모션 | "개원 3주년 이벤트" |
| `equipment` | 신규 장비/시설 | "3D CT 장비 도입 안내" |
| `staff` | 인사/채용 | "새 원장님 소개" |
| `general` | 일반 공지 | 기타 공지사항 |

**생성 흐름:**
```
템플릿 선택 → 필수 정보 입력 (UI 폼)
→ AI가 공지문 생성 → 병원 사진 첨부 (직접 업로드)
→ 미리보기 → 발행
```

**입력 폼 예시 (holiday 템플릿):**
```typescript
interface NoticeHolidayInput {
  holidayName: string;       // "설 연휴"
  closedFrom: string;        // "2026-01-28"
  closedTo: string;          // "2026-01-30"
  resumeDate: string;        // "2026-01-31"
  emergencyContact?: string; // 응급 연락처 (선택)
  additionalNote?: string;   // 추가 안내사항 (선택)
}
```

**생성 결과 예시:**
```
제목: 하얀치과 설 연휴 진료 안내

안녕하세요, 하얀치과입니다.
설 연휴 기간 휴진 안내드립니다.

📅 휴진 기간: 1월 28일(수) ~ 1월 30일(금)
📅 정상 진료: 1월 31일(토)부터

연휴 기간 중 치과 응급 상황 시 [응급 연락처]로 연락 부탁드립니다.
새해 복 많이 받으세요!

- 하얀치과 드림
```

**공지글 특징:**
- AI 이미지 생성 불필요 → 병원 로고/사진 직접 첨부
- 짧은 글 (500자 미만) → 검색허용 해제 권장 (블로그 지수 보호)
- 키워드 SEO 불필요 → 기존 환자 대상 안내 목적

### 10-3. 임상글 (Clinical Case)

실제 병원에서 촬영한 임상 사진을 활용한 시술 사례 포스팅.

**임상글 생성 흐름:**
```
임상 사진 업로드 (전/후, 과정)
→ 시술 정보 입력 (UI 폼)
→ AI가 사진 분석 + 본문 생성
→ 의료법 준수 자동 검증
→ 원장 최종 검토/승인
→ 발행
```

**입력 폼:**
```typescript
interface ClinicalCaseInput {
  // 시술 정보
  procedureType: string;        // "임플란트" | "교정" | "미백" | "충치치료" 등
  procedureDetail?: string;     // "상악 좌측 제1대구치 임플란트"
  duration?: string;            // 시술 기간 "3개월"

  // 임상 사진 (직접 업로드)
  photos: {
    type: 'before' | 'after' | 'process' | 'xray';
    file: File;
    caption?: string;           // 사진 설명 (선택)
  }[];

  // 환자 정보 (익명)
  patientAge?: string;          // "40대"
  patientGender?: string;       // "여성"
  chiefComplaint?: string;      // "어금니 상실로 저작 곤란"

  // 글 옵션
  tone: ToneType;               // 어투 선택
  useResearch: boolean;         // 논문 인용 여부
  includePrice: boolean;        // 비용 정보 포함 여부 (의료법 주의)
}
```

**AI 본문 생성 구조:**
```
1. 도입부: 환자 고민/증상 공감 (익명)
   "어금니가 빠져서 식사가 불편하셨던 40대 여성 환자분의 사례입니다."

2. 진단: 상태 설명 (X-ray/사진 기반)
   "파노라마 촬영 결과, 상악 좌측 제1대구치 부위의 골밀도가..."
   [X-ray 사진]

3. 치료 계획: 시술 방법 설명
   "임플란트 식립 후 3개월간 골유착 기간을 거쳐..."

4. 시술 과정: 단계별 설명 + 임상 사진
   [시술 전 사진] → [과정 사진] → [시술 후 사진]

5. 결과 & 관리법: 시술 결과 + 사후 관리 안내
   "시술 후 정기적인 검진과 올바른 구강 위생 관리가 중요합니다."

6. (옵션) 논문 인용: 관련 연구 결과
```

### 10-4. 임상 사진 처리 규칙

**개인정보 보호 (필수):**
- [ ] 환자 얼굴 자동 모자이크/크롭 (구강 부위만 노출)
- [ ] 환자 이름/차트번호 제거
- [ ] 환자 동의서 확인 플래그 (`patientConsent: boolean`)
- [ ] 동의서 미확인 시 발행 차단

**이미지 전처리:**
```typescript
interface PhotoProcessing {
  autoAnonymize: boolean;    // 얼굴 자동 모자이크
  addWatermark: boolean;     // 병원 로고 워터마크
  adjustBrightness: boolean; // 밝기/대비 자동 보정
  resizeForBlog: boolean;    // 블로그 최적 크기 조정 (가로 860px)
}
```

**사진 배치 패턴:**
```
[Before → After] 가로 나란히 비교 (가장 임팩트 있음)
[단계별 과정]    시간순으로 세로 배치
[X-ray + 구강]   진단 사진 → 임상 사진 순서
```

임상글 발행 전 자동으로 의료법 위반 여부 체크:

## 11. 멀티 플랫폼 배포 (SNS 확장)

네이버 블로그 외에 페이스북, 인스타그램, 쓰레드에도 콘텐츠를 배포.

### 11-1. 플랫폼별 특성

| 항목 | 네이버 블로그 | 인스타그램 | 페이스북 | 쓰레드 |
|------|-----------|----------|---------|--------|
| 글자수 | 1,000자+ | 2,200자 제한 | 63,206자 | 500자 제한 |
| 이미지 | 3~7장 (세로 배치) | 1~10장 (캐러셀) | 1~10장 | 1장 (선택) |
| 이미지 비율 | 자유 | **1:1 / 4:5** | 자유 | 자유 |
| 해시태그 | 10개 미만 | **최대 30개** (15~20개 권장) | 3~5개 | 5~10개 |
| 링크 | 본문 내 가능 | **본문 링크 불가** (프로필만) | 본문 내 가능 | 본문 내 가능 |
| 톤 | 정보성/전문적 | **비주얼 중심/감성적** | 커뮤니티/공유형 | **짧고 캐주얼** |
| 배포 방식 | Playwright | **Graph API** | **Graph API** | **Threads API** |
| 자동화 난이도 | 중간 (브라우저) | 쉬움 (공식 API) | 쉬움 (공식 API) | 쉬움 (공식 API) |

### 11-2. 원본 → 플랫폼별 변환 전략

하나의 콘텐츠를 작성하면 각 플랫폼에 맞게 자동 변환:

```
원본 콘텐츠 (네이버 블로그용 1,000자+ 정보글)
├→ 인스타그램: 핵심 요약 + 캐러셀 이미지 + 해시태그
├→ 페이스북: 중간 길이 요약 + 대표 이미지 + 링크
└→ 쓰레드: 핵심 한 줄 + 호기심 유발 + 블로그 링크 유도
```

```typescript
interface ContentDistribution {
  // 원본 (네이버 블로그)
  source: {
    title: string;
    body: string;        // 전체 본문
    images: ImageFile[];
    blogUrl?: string;    // 발행 후 URL
  };

  // 플랫폼별 변환 옵션
  platforms: {
    instagram: boolean;
    facebook: boolean;
    threads: boolean;
  };

  // 배포 스케줄
  schedule: {
    blog: Date;           // 블로그 먼저 발행
    snsDelay: number;     // 블로그 발행 후 N분 뒤 SNS 배포 (기본: 30분)
  };
}
```

### 11-3. 플랫폼별 변환 규칙

**인스타그램:**
```typescript
interface InstagramPost {
  images: {
    files: File[];          // 1~10장, 1:1 또는 4:5 비율로 자동 크롭
    type: 'single' | 'carousel';
  };
  caption: string;          // 본문 요약 (2,200자 이내)

```

- 이미지를 **1:1 또는 4:5 비율**로 자동 크롭/리사이즈
- 정보성 글 → **캐러셀(슬라이드)** 형태가 저장/공유율 높음
- 마지막 슬라이드에 "자세한 내용은 프로필 링크 확인" CTA

  ...

### 11-7. 배포 플랫폼 사용자 설정

배포 플랫폼은 **2단계로 사용자가 제어**:

**① 글로벌 설정 (관리자 설정 페이지)**
사용할 플랫폼을 미리 연동/활성화. 연동하지 않은 플랫폼은 선택지에 노출되지 않음.

```typescript

```

**② 글별 선택 (글 작성/캘린더 항목마다)**
활성화된 플랫폼 중에서 해당 글을 어디에 배포할지 선택.

```
┌─────────────────────────────────────┐
│  📢 배포 플랫폼 선택                    │
│                                     │
│  ☑️ 네이버 블로그 (기본)                │
│  ☑️ 인스타그램    @hayan_dental       │

│  ⚠️ 페이스북 미선택 시 해당 플랫폼에는    │
│     발행되지 않습니다.                  │
└─────────────────────────────────────┘
```

**글 유형별 기본 플랫폼 프리셋:**

| 글 유형 | 기본 선택 플랫폼 | 변경 가능 |
|--------|-------------|---------|
| 정보성 | 블로그 + 인스타 + 쓰레드 | O |
| 홍보성 | 블로그 + 인스타 + 페이스북 + 쓰레드 | O |
| 공지글 | 블로그만 | O |
| 임상글 | 블로그 + 인스타 | O |

프리셋은 사용자가 변경 가능하며, 변경된 프리셋은 이후 캘린더 자동 생성 시 반영.

## 12. 콘텐츠 캘린더 & 스케줄 관리

AI가 주간/월간 콘텐츠 계획을 제안하고, 사용자가 컨펌하면 그 스케줄대로 자동 발행.

### 12-1. 전체 흐름

```
AI가 주간 콘텐츠 캘린더 생성 (주제, 키워드, 발행일시, 플랫폼)
→ 관리자 UI에서 캘린더 형태로 미리보기
→ 사용자가 수정/승인/거절
→ 승인된 항목만 예약 큐에 등록
→ 발행 시간 도달 시 자동 실행 (글 생성 → 발행)
→ 발행 결과 알림 (성공/실패)
```

**AI 캘린더 생성 시 고려 사항:**
- 같은 키워드 연속 발행 방지 (최소 2주 간격)
- 정보성 : 홍보성 = 2:1 비율 자동 배분
- 계절/시기에 맞는 주제 자동 반영 (예: 여름 → "냉각 음료와 치아 시림", 연말 → "연말 치과 검진")

- 하루 최대 3건 발행 제한 준수
- 주말/공휴일 발행 여부 설정 가능

승인된 항목이 발행 시간에 도달하면 자동 실행:

```
[스케줄러] 매 5분마다 체크
→ 발행 시간 도달한 approved 항목 발견
→ status: 'generating' (글 생성 시작)
  → 키워드 조사 (검색량/경쟁도)
  → AI 본문 생성 (옵션 적용: 어투, 논문, 팩트체크)
  → AI 이미지 생성 (나노바나나 2)
→ status: 'review' (임상글만 - 원장 승인 대기)
→ status: 'scheduled' (발행 준비 완료)
→ status: 'publishing' (발행 실행)
  → 네이버 블로그 발행 (Playwright, 체류시간 확보)
  → 30분 대기
  → SNS 순차 배포 (Graph API / Threads API)
→ status: 'published' (완료, URL 저장)
→ 관리자에게 알림 (앱 내 알림 / 푸시)
```

## 13. 에디터 입력 자동화 - 체류 시간 확보 전략

> 출처: 유튜버 '마토' 영상 (2,200개 글 자동화 포스팅, 저품질 없이 운영)

### 핵심 원칙
네이버는 **"사람이 할 수 없는 속도로 행동하는가?"**를 실시간 체크한다.
체류 시간 확보 = **네이버 로봇을 속이기 위한 인간다운 행동 패턴 모사**

### 13-1. 복사 붙여넣기 절대 금지

| 방식 | 네이버 인식 | 결과 |
|------|----------|------|
| Ctrl+V (복사 붙여넣기) | 0.1초에 수천자 입력 → 기계적 등록 | **저품질/누락** |
| 한 글자씩 타이핑 | 키보드 신호와 동일 → 사람이 작성 | **안전** |

### 13-2. 타이핑 시뮬레이션 (Human-like Typing)

```typescript
// 글자 하나당 랜덤 딜레이 (0.01~0.05초)
async function humanType(page: Page, selector: string, text: string) {
  await page.click(selector);
  for (const char of text) {
    await page.keyboard.type(char, { delay: 0 });
    // 글자마다 10~50ms 랜덤 딜레이
    await page.waitForTimeout(10 + Math.random() * 40);
  }
}

// 문단 입력 (문단 사이 1~3초 휴식)
async function typeParagraph(page: Page, paragraphs: string[]) {
  for (const para of paragraphs) {
    await humanType(page, '.se-text-paragraph', para);
    await page.keyboard.press('Enter');
    // 문단 사이 사람이 생각하는 시간 시뮬레이션
    await page.waitForTimeout(1000 + Math.random() * 2000);
  }
}
```

- 2,000자 글 → 타이핑에 약 1~2분 소요 (자연스러운 페이지 체류시간 확보)
- **고정 딜레이 금지** → 반드시 랜덤 범위 사용 (매번 똑같으면 매크로 의심)

### 13-3. 단계별 의도적 지연 (Wait 로직)

각 동작 단계 사이에 반드시 대기 시간 삽입:

```typescript

  titleToBody:   { min: 1500, max: 2500 },   // 제목→본문 이동
  imageUpload:   { min: 2000, max: 4000 },   // 이미지 업로드 후
  beforeSave:    { min: 1500, max: 3500 },   // 저장 버튼 누르기 전
  afterSave:     { min: 3000, max: 5000 },   // 저장 완료 후
};

function randomDelay(config: { min: number; max: number }): number {
  return config.min + Math.random() * (config.max - config.min);
}
```

| 글쓰기 창 진입 후 | 2~3초 | 페이지 로딩/레이아웃 인식 시뮬레이션 |

### 13-4. iframe 처리 주의

네이버 에디터 본문은 **iframe 내부**에 존재:
- 본문 입력: iframe 안으로 진입
- 저장/카테고리 등 버튼: iframe 밖
- **iframe 전환 속도가 너무 빠르면 버튼 클릭 실패**
- 전환 시 반드시 1.5~3.5초 대기

### 13-5. 전체 포스팅 1건의 목표 소요 시간

```
페이지 진입 (2~3초)
→ 팝업 처리 (1~2초)
→ 카테고리 선택 (1~2초)
→ 제목 타이핑 (5~15초)
→ 본문 이동 (1.5~2.5초)
→ 본문 타이핑 (60~120초, 글자수에 따라)
→ 이미지 삽입 x 3~5장 (각 3~5초)
→ 해시태그 입력 (5~10초)
→ 최종 확인 대기 (1.5~3.5초)
→ 발행 클릭 (1초)
→ 발행 완료 대기 (3~5초)
────────────────────
총 약 5~10분 / 1건
```

- 빠르게 많이 올리는 것보다 **1건을 사람처럼 천천히 올리는 것이 핵심**
- 이 방식으로 2,200개 글을 저품질 없이 운영 가능 (마토 사례)
